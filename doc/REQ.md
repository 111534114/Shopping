# 點飲料點餐系統 — 需求規格文件（REQ.md）

## 一、系統概述

本系統旨在建立一個提供飲品訂購的線上點餐平台，讓顧客能方便地於網站上瀏覽菜單、選擇飲品與客製化選項、線上付款並追蹤訂單狀態，同時讓店員與管理員能於後台進行訂單處理與菜單管理，以提升整體營運效率。

---

## 二、系統角色與功能

| 角色 | 功能描述 |
|------|-----------|
| 顧客 (Customer) | 瀏覽菜單、選擇飲品、設定客製化（甜度、冰塊）、線上付款、查看訂單狀態、取餐時出示訂單資訊。 |
| 店員 (Staff) | 接收新訂單通知、更新訂單狀態（準備中 / 已完成待取餐）、確認顧客取餐。 |
| 管理員 (Admin) | 管理菜單（新增/修改/刪除）、查看銷售報表、維護系統資料。 |

---
## 七、使用情境（Use Case）

| 角色 | 使用情境 | 系統反應 |
|------|-----------|-----------|
| 顧客 | 點選「珍珠奶茶」→ 選擇「微糖、去冰」→ 加入購物車 | 顯示購物車更新內容 |
| 顧客 | 點選「結帳」並選擇  | 轉導至支付頁面並建立訂單 |
| 店員 | 登入後台 → 查看新訂單 | 顯示新訂單詳情 |
| 店員 | 更新訂單狀態為「準備中」 | 顧客前台顯示狀態同步更新 |
| 管理員 | 編輯飲品價格 | 後台更新資料庫並回傳成功訊息 |

## 三、功能需求

### （1）前台系統功能

| 編號 | 功能名稱 | 功能描述 |
|------|-----------|-----------|
| F1 | 菜單瀏覽 | 顧客可查看所有飲品品項及其價格、圖片、介紹。 |
| F2 | 客製化選項 | 顧客可選擇飲品甜度、冰塊、加料等選項。 |
| F3 | 購物車 | 顧客可將選擇的商品加入購物車，修改數量或刪除品項。 |
| F4 | 結帳流程 | 顧客可透過信用卡或 LINE Pay 完成付款。 |
| F5 | 訂單追蹤 | 顧客可查看訂單狀態（已下單、準備中、已完成待取餐）。 |

---

### （2）後台系統功能

| 編號 | 功能名稱 | 功能描述 |
|------|-----------|-----------|
| B1 | 新訂單通知 | 店員於後台即時接收顧客下單通知。 |
| B2 | 訂單狀態管理 | 店員可更新訂單狀態，系統即時同步顧客端。 |
| B3 | 訂單查詢 | 店員可查詢歷史訂單紀錄與詳細內容。 |
| B4 | 菜單管理 | 管理員可新增、編輯或刪除飲品項目。 |
| B5 | 銷售報表 | 管理員可生成日/週/月銷售報表與統計圖表。 |

---

## 四、非功能性需求

| 類別 | 描述 |
|------|------|
| 效能 | 系統應能在 2 秒內載入主要頁面；訂單建立時間不超過 3 秒。 |
| 可用性 | 系統需支援電腦與手機裝置瀏覽。 |
| 安全性 | 所有交易資料需加密傳輸（HTTPS），並記錄訂單與支付資訊。 |
| 擴充性 | 系統架構採前後端分離，支援未來功能模組擴充。 |

---

## 五、系統互動流程（對應 SA.md）

| 流程名稱 | 說明 |
|-----------|------|
| P1 | 顧客進入網站 → 瀏覽菜單 → 選擇飲品 → 加入購物車 → 結帳下單 → 後端儲存訂單 → 顯示訂單完成頁面。 |
| P2 | 顧客可於前台查看訂單狀態，後端與資料庫同步更新狀態（準備中 / 已完成）。 |
| P3 | 顧客取餐時出示訂單資訊，店員確認後交付飲品並完成訂單流程。 |

---

## 六、系統架構說明（對應 SD.md）

### 架構層級：

1. **前台（Frontend）**
    - 使用 HTML / CSS / JavaScript 建構。
    - 與後端透過 RESTful API 溝通。
    - 提供顧客點餐介面與狀態顯示。

2. **後台（Backend）**
    - 使用 Spring Boot / Node.js 等框架實作。
    - 提供訂單、菜單、報表 API。
    - 管理員與店員介面整合於後台管理系統中。

3. **資料庫（Database）**
    - 使用 MySQL 儲存菜單與訂單資料。
    - 資料表：
        - `menu_items`（飲品資訊）
        - `orders`（訂單資料）
        - `order_items`（訂單細項）
        - `users`（顧客 / 店員 / 管理員帳號）
