# 點飲料點餐系統 — 需求規格文件（REQ.md｜Python 版）

## 一、系統概述

本系統旨在建立一個線上點飲料平台，讓顧客可透過瀏覽器完成飲品瀏覽、客製化選項選擇、線上付款與訂單追蹤；同時提供店員與管理員後台系統，以利接單、製作、菜單管理與銷售分析。

後端系統採用 **Python（Flask 或 FastAPI）** 實作 RESTful API，前後端分離設計，提升系統可維護性與擴充性。

---

## 二、系統角色與職責

| 角色           | 職責說明                                      |
| ------------ | ----------------------------------------- |
| 顧客（Customer） | 瀏覽菜單、選擇飲品與客製化（甜度、冰塊）、加入購物車、付款、查詢訂單狀態、到店取餐 |
| 店員（Staff）    | 查看新訂單、更新訂單狀態（準備中 / 已完成待取餐）、核對顧客取餐         |
| 管理員（Admin）   | 菜單管理（新增 / 編輯 / 刪除）、系統資料維護          |

---

## 三、功能性需求

### （一）前台系統功能（Customer）

| 編號 | 功能名稱  | 功能描述                       |
| -- | ----- | -------------------------- |
| F1 | 菜單瀏覽  | 顯示所有飲品品項、價格、圖片與簡介          |
| F2 | 客製化選項 | 可選擇甜度、冰塊、加料等參數             |
| F3 | 購物車   | 加入 / 修改 / 刪除商品，顯示總金額       |
| F4 | 結帳流程  | 現金        |
| F5 | 訂單追蹤  | 顯示訂單狀態（已下單 / 準備中 / 已完成） |

### （二）後台系統功能（Staff / Admin）

| 編號 | 功能名稱   | 功能描述             |
| -- | ------ | ---------------- |
| B1 | 新訂單  | 店員可即時或定期取得新訂單列表  |
| B2 | 訂單狀態管理 | 店員可更新訂單狀態，前台即時同步 |
| B3 | 訂單查詢   | 查詢歷史訂單與訂單明細      |
| B4 | 菜單管理   | 管理員可新增、編輯、刪除飲品   |


---

## 四、非功能性需求

| 類別  | 說明                      |
| --- | ----------------------- |
| 效能  | 主要頁面 2 秒內載入，訂單建立不超過 3 秒 |
| 可用性 | 支援桌機與手機（RWD）            |
| 安全性 | 採用 HTTPS 傳輸，避免明碼傳輸敏感資料  |
| 擴充性 | 前後端分離，後端 API 模組化設計      |

---

## 五、系統流程（Use Case 摘要）

| 流程代號 | 說明                               |
| ---- | -------------------------------- |
| P1   | 顧客瀏覽菜單 → 客製化 → 加入購物車 → 結帳 → 建立訂單 |
| P2   | 店員查看新訂單 → 更新狀態 → 顧客端同步顯示         |
| P3   | 顧客到店取餐 → 店員確認 → 訂單完成             |

---

## 六、技術限制與假設

* 後端以 **Python Flask 或 FastAPI** 實作
* 資料庫使用 MySQL
* 通知機制 MVP 階段以「前端輪詢（Polling）」實作
