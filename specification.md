# 雞娃娃 (Chihuahua) 飲品數位平台規格書

## 1. 系統總覽與目標

本文件旨在定義「雞娃娃」品牌之數位點餐與行銷平台的系統規格。

### 1.1. 核心任務
此數位平台的主要任務是作為品牌營銷和銷售的數位化門戶，以達成以下目標：
- **促進銷售**：提供顧客便利的線上菜單瀏覽與點餐功能。
- **強化品牌**：透過系統介面呈現「萌」系視覺風格，突顯可愛的「雞娃娃」吉祥物，傳遞「活力、療癒、幸福感」的品牌定位。
- **自動化行銷**：執行並追蹤「金蛋幸運日」、「雞娃娃扭蛋機」等行銷活動，提升顧客回訪率與客單價。

### 1.2. 目標客群
主要面向追求新潮、注重社群分享的年輕族群，如高中生、大學生及上班族。

---

## 2. 系統架構 (System Architecture)

### 2.1. 系統分層
本系統採用前後端分離架構。

- **前端 (Presentation Layer)**：負責所有用戶介面的渲染與互動。使用響應式網頁技術 (HTML/CSS/JS) 實現，確保在不同裝置上均有良好體驗。
- **後端 (Service Layer)**：負責處理所有業務邏輯、資料存取與外部服務整合。作為一個強大的後台，提供 API 服務並連接資料庫。

### 2.2. 技術選用
- **後端框架**：**Spring Boot**
- **前端技術**：**HTML, CSS, JavaScript**
- **資料庫**：關聯式資料庫 (如 MySQL 或 PostgreSQL)
- **版本控制**：Git

### 2.3. 系統比喻
此數位系統如同為「雞娃娃」打造的**「數位雞舍」**。前台是吸引顧客的漂亮展示區，後台則是自動化的飼料機與孵蛋器，確保物料供應無虞，並能精準派發優惠，讓顧客（雞群）不斷回訪。

## 3. 功能模組 (Functional Modules)

### 3.1. 產品資料管理模組 (Product Management)
此為系統最重要模組，若移除則所有點餐與行銷功能皆會失效。
- **職責**：管理飲品及周邊商品的資訊。
- **功能**：
    - 新增、讀取、更新、刪除 (CRUD) 商品。
    - 管理商品名稱、核心賣點、價格、庫存、行銷文案（支援多國語言）。
    - 設定飲品的客製化選項（如甜度、冰度、加料）。

### 3.2. 訂單處理模組 (Order Processing)
- **職責**：處理顧客的點餐流程。
- **功能**：
    - 提供購物車功能。
    - 建立訂單，並進行庫存檢查。
    - 計算訂單總金額。
    - 應用優惠券或活動折扣。

### 3.3. 會員與認證模組 (User & Auth)
- **職責**：管理顧客與後台管理員的資料與權限。
- **功能**：
    - 顧客註冊與登入。
    - 管理員身份驗證與權限控管。
    - 記錄顧客歷史訂單與消費金額。
    - 管理顧客持有的優惠券或獎勵。

### 3.4. 行銷活動模組 (Marketing & Promotions)
- **職責**：支援並自動化品牌的行銷活動。
- **功能**：
    - **金蛋幸運日**：處理週一回訪顧客的打卡與優惠（如：免費加布丁）。
    - **雞娃娃扭蛋機**：當訂單滿足特定門檻（如：滿 120 元）時，觸發扭蛋抽獎機制。獎品可為折價券或品牌周邊。
    - **拍照打卡挑戰**：處理用戶生成內容 (UGC) 的上傳與追蹤。

## 4. API 設計

為強化品牌連結，API 命名採用與「雞娃娃」及「金蛋」意象相關的主題。

| 方法 | API 路徑 | 功能描述 | 命名理由 |
| :--- | :--- | :--- | :--- |
| `GET`  | `/chick/menu` | 查詢所有可販售的產品列表。 | **小雞菜單**：呼應可愛的小雞吉祥物。 |
| `GET`  | `/chick/product/{id}` | 查詢單一產品的詳細資訊與庫存。 | - |
| `POST` | `/egg/hatch` | 建立一筆新訂單。 | **孵化訂單**：呼應「幸福就在嘴裡孵化」的文案。 |
| `GET`  | `/treasure/spin` | 觸發扭蛋機活動。 | **寶藏旋轉**：呼應「金蛋象徵驚喜與幸福」的寶藏概念。 |
| `POST` | `/nest/luckyday` | 處理「金蛋幸運日」的顧客打卡。 | **巢穴幸運日**：處理週一回訪的專屬活動。 |
| `GET`  | `/user/loyalty` | 查詢會員持有的優惠券與獎勵。 | - |

## 5. 產品規格 (Product Catalog)

### 5.1. 金蛋奶茶 (招牌)
- **口味建議**：焦糖布丁 + 濃郁奶茶 + QQ珍珠，口感豐富有層次。
- **命名靈感**：雞蛋 → 金蛋，呼應品牌形象，象徵珍貴與驚喜。
- **推薦文案**：金光閃閃的滑嫩布丁在奶茶裡化開，像發現了雞娃娃的寶藏！

### 5.2. 小雞抹茶拿鐵
- **口味建議**：抹茶牛奶 + 白玉珍珠，抹茶微苦回甘，溫潤順口。
- **命名靈感**：抹茶綠像小雞蛋殼的顏色，白玉珍珠宛如蛋白。
- **推薦文案**：綿密抹茶搭配Q彈白玉，就像雞娃娃窩裡的小幸福！

### 5.3. 咕咕奶茶
- **口味建議**：經典紅茶奶茶基底，可搭配黑糖珍珠、布丁或椰果。
- **命名靈感**：「咕咕」是小雞啼叫聲，可愛俏皮；「奶霸」象徵鎮店之寶。
- **推薦文案**：「咕咕叫的招牌奶茶來啦！」茶香與奶香交織，每一口都像小雞啾啾般活潑可愛。

### 5.4. 咕韻黑糖鮮奶
- **口味建議**：濃郁黑糖 + 香醇鮮奶 + QQ珍珠，溫潤療癒。
- **命名靈感**：「咕韻」呼應「咕咕」小雞元素，並增添一份古早味韻味。
- **推薦文案**：濃郁黑糖熬出的溫潤甜香，遇上香醇鮮奶，喝一口就像被小雞娃娃輕輕擁抱。

### 5.5. 百香椰啾綠
- **口味建議**：百香果 + 清新綠茶 + Q彈椰果，活潑有層次。
- **命名靈感**：「啾綠」呼應小雞啾啾的俏皮感，結合綠茶。
- **推薦文案**：一口酸甜，一口Q彈，還有小雞啾啾的清爽快樂感！

## 6. 未來擴充性

此系統設計已為未來的業務擴展預留空間：

1.  **國際化 (i18n)**：產品文案與系統內容已考慮多語言支持，為進入海外市場（如韓國）做準備。
2.  **產品類別擴充**：扭蛋機獎品包含「周邊」，暗示產品模組可擴充至銷售非飲品的實體商品（如雞娃娃玩偶）。
3.  **活動多樣化**：已設計多種活動模組，未來可輕易增加季節限定、聯名合作等新促銷規則。
