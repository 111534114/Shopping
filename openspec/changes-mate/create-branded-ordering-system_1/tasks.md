# 任務清單：建立品牌化點餐系統

- [ ] **Phase 1: 基礎架構設置**
    - [ ] 1.1 初始化 Spring Boot 專案，並加入 `web`, `data-jpa`, `mysql-connector-j` 依賴。
    - [ ] 1.2 設定資料庫連線，並建立初始的資料表 (`products`, `orders`, `users`)。
    - [ ] 1.3 建立基礎的 RESTful API 端點結構 (Controller, Service, Repository)。
    - [ ] 1.4 建立前端專案 (Vue.js/React)，並完成與後端的 API 連線測試。

- [ ] **Phase 2: 核心點餐功能開發**
    - [ ] 2.1 **[後端]** 開發 `GET /api/products` 端點，回傳所有飲品列表。
    - [ ] 2.2 **[前端]** 開發菜單頁面，渲染從後端獲取的飲品列表。
    - [ ] 2.3 **[前端]** 開發購物車功能，允許使用者新增、移除、修改品項。
    - [ ] 2.4 **[後端]** 開發 `POST /api/orders` 端點，接收前端訂單資訊並存入資料庫。
    - [ ] 2.5 **[前端]** 開發結帳頁面，串接支付流程並在成功後顯示訂單完成訊息。

- [ ] **Phase 3: 品牌與行銷功能開發**
    - [ ] 3.1 **[前端]** 根據品牌風格，設計並實作全站的 UI 元件（按鈕、色彩、字體）。
    - [ ] 3.2 **[前端]** 在首頁和重要頁面加入「雞娃娃」吉祥物視覺元素。
    - [ ] 3.3 **[後端]** 設計並建立 `marketing_rules` 資料表。
    - [ ] 3.4 **[後端]** 在訂單建立邏輯中，加入檢查是否觸發「扭蛋機」活動的判斷。
    - [ ] 3.5 **[前端]** 開發「扭蛋機」互動介面。
 。

- [ ] **Phase 5: 測試與驗收**
    - [ ] 5.1 撰寫後端單元測試與整合測試。
    - [ ] 5.2 執行端對端 (E2E) 測試，確保點餐流程順暢。
    - [ ] 5.3 進行使用者驗收測試 (UAT)。
