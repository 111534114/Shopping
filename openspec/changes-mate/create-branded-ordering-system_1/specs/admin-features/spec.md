# 規格：管理功能 (Admin Features)

## ADDED Requirements

### #### Requirement: `admin-features-1` 店員能夠接收並管理新訂單
- **As a** 店員
- **I want to** 在後台看到新進的訂單並更新其狀態
- **So that** 我可以開始準備飲品並讓顧客了解進度。

#### Scenario: 處理新訂單
- **Given** 我是已登入後台的店員，且有一筆新訂單進來
- **When** 我在訂單列表頁點擊該筆訂單的「開始製作」按鈕
- **Then** 該訂單的狀態應更新為「準備中」，且顧客端的前台介面會同步看到此狀態更新。

### #### Requirement: `admin-features-2` 管理員能夠管理菜單品項
- **As a** 管理員
- **I want to** 新增、修改或下架網站上的飲品
- **So that** 確保菜單內容的即時性與正確性。

#### Scenario: 新增飲品
- **Given** 我是已登入後台的管理員
- **When** 我在菜單管理頁面填寫「百香椰啾綠」的名稱、價格 (60元) 和介紹，並點擊「儲存」
- **Then** 顧客在前台的菜單上應該能看到新的「百香椰啾綠」品項。

### #### Requirement: `admin-features-3` 管理員能夠查看銷售報表
- **As a** 管理員
- **I want to** 查看指定時間範圍內的銷售總額與熱銷品項
- **So that** 我可以了解營運狀況並制定商業決策。

#### Scenario: 查看月報表
- **Given** 我是已登入後台的管理員
- **When** 我在報表頁面選擇「上個月」並點擊「生成報表」
- **Then** 系統應顯示上個月的總銷售金額，以及銷量前三名的飲品排行。
